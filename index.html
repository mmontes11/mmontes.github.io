GeoTourism is a management platform for places and routes of touristic interest

The platform has a RESTful webservice implemented with Spring that exposes operations related with places and routes and manages the different types of users. The data is stored in a PostgreSQL database and is accessed via Hibernate Spatial

The platform also consumes external APIs such as: Amazon Web Services(S3), Google Maps, Facebook ...

The client side is a SPA developed with AngularJS. It consumes the RESTful webservice and represents the spatial data using a Leaflet map. In order to achieve a better user experience, I used Angular Material which allows you to make a responsive frontend using the material design principles

The aim of this project is developing a Big Data module for a tourism CMS where the administrator can see statistical information of his website

One first part of this module is a high performance NodeJS RESTful webservice which receives metrics from the client side and stores them in a MongoDB database

In order to process and transform this big amount of data we used an ETL(Extraction Transform Load). This process is executed every night and stores the processed data in another MongoDB

Once the data is processed, the admin can consult this data using a dashboard implemented with BackboneJS. It displays the information using different types of charts.
